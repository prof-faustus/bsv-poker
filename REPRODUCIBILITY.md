# Reproducibility

> **⚠ Peer-to-peer update.** bsv-poker is now **fully peer-to-peer** — there is **NO central relay or
> indexer server** (`apps/relay-go` and `apps/indexer-go` are deleted). Where this document still says
> "relay"/"indexer", read it as the gossip transport (`packages/adapters/src/p2p-transport.ts`) and the
> player's own local node (`tools/local-node.ts`): the SAME signed envelope protocol carried directly
> between peers, with discovery as a gossiped directory. The same model runs on regtest, testnet, and
> real BSV. Authoritative description: [`P2P_MODEL.md`](./P2P_MODEL.md).

Determinism is a security property here: the relay/indexer are never the source of truth, so a
reconnecting or disputing client must recompute the same bytes from the same inputs. "Zero
fabrication" is a hard rule — every committed vector/byte-size is generated by running code.

## What is reproducible

- **Test vectors** under `spec/vectors` and the byte-size schedules: regenerated by
  `pnpm reproduce` (`tools/reproduce.ts`), which exits non-zero on any mismatch (core §14.5).
- **State hashes**: two honest clients hashing the same logical state produce identical bytes
  (`serialize.ts` canonical encoding); proven by the convergence e2es.
- **Transcript rebuild**: `rebuildHand` over the indexer's records reproduces the live state hash
  byte-for-byte (`tools/reconnect-e2e.ts`, `tools/validating-indexer-e2e.ts`).
- **Parser identity**: `serializeParsedTx(parseTxWire(b).tx) === b` for canonical `b` (fuzz-proven).
- **Canonical signed message**: the Go indexer reconstructs the TS-signed message byte-for-byte
  (pinned vector `INV-IXV-0`).

## How determinism is enforced

- The engine core is **pure**: no I/O, no `Date.now()`, no randomness inside the deterministic path
  (randomness enters only as committed entropy in the shuffle, and the CSPRNG for fresh secrets).
- All security/replay-relevant bytes go through the canonical serializer; floats are banned.
- `pnpm trace` checks every `REQ-*` maps to code and a passing test, so a behaviour cannot drift
  without a failing trace.

## Commands

```
pnpm reproduce     # regenerate every committed vector; non-zero on any mismatch
pnpm trace         # requirements -> code -> test traceability
pnpm ci            # includes both, plus the full test + fuzz + go suites
```

## Toolchain pinning

Node ≥ 24 (runs TS directly), pnpm 9, Go ≥ 1.24. The pure-TS SHA-256 (`protocol-types/sha256.ts`)
removes any dependence on a platform crypto implementation for the deterministic core, so the same
bytes are produced in Node and in the browser bundle.
